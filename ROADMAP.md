# Roadmap

Zoe is being developed in public. The roadmap below describes the next milestones after the v0.1 Public Preview.

> **Status legend**: ✅ shipped · 🟡 in progress · ⚪ planned

## Current

### v0.1 Public Preview ✅

- [x] GitHub Login (one-command OAuth)
- [x] Project Intelligence (auto-scan)
- [x] AI Chat with streaming
- [x] File reading, writing, and targeted editing
- [x] Execution planning
- [x] Slash commands
- [x] Session persistence

## Next

### v0.2 Better Terminal UI 🟡

- [ ] Inline diffs in the terminal
- [ ] Multi-pane layout
- [ ] Theme support
- [ ] Streaming tool output
- [ ] Resumable sessions

### v0.3 Project Intelligence 🟡

- [ ] Semantic search across the codebase
- [ ] Dependency graph visualisation
- [ ] Code hotspot analysis
- [ ] Multi-file context retrieval
- [ ] Configurable ignore patterns

### v0.4 Zoe Cloud 🟡

- [ ] Cross-device session sync
- [ ] Shareable workspaces
- [ ] Team projects
- [ ] Audit log
- [ ] Project-level permissions

### v1.0 Stable ⚪

- [ ] Public API freeze
- [ ] Plugin SDK
- [ ] Enterprise SSO
- [ ] Stable protocol for tools
- [ ] Long-term support commitment

## Future ideas

> These are explorations, not commitments. They may change based on feedback.

- [ ] **Skills** — opt-in extension packs for specific frameworks (React, Django, Rails…)
- [ ] **MCP** — Model Context Protocol support so Zoe can use any MCP server
- [ ] **Project Memory** — long-term knowledge per project that survives across sessions
- [ ] **Cloud Sync** — keep your settings and history in sync across machines
- [ ] **Plugin System** — community-built tools and integrations
- [ ] **Self-hosted mode** — run Zoe against your own InsForge-compatible backend
- [ ] **Team workspaces** — shared memory and project context for engineering teams

---

## How this roadmap works

- We ship in short cycles. A milestone is "done" when its checklist is empty, not when a calendar date passes.
- Items can move between milestones based on user feedback and technical discoveries.
- If something here is important to you, open a GitHub issue with the `roadmap` label and tell us why.
